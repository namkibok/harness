# UX Experience

## 지시

1. **입력 정리** — 목표 사용자, 핵심 과업, 제약(플랫폼, 오프라인, 법규)을 `_workspace/00_input/`에 요약하거나 사용자 메시지에서 추출한다.
2. **시나리오** — 해피패스 1개 + 에러/취소/권한 부족 등 에지 2개 이상을 짧은 스토리로 쓴다.
3. **플로 문서** — `_workspace/05_ux_flow.md`에 다음을 포함한다:
   - 화면·단계 **ID**(예: `SCR-LOGIN-01`)
   - 단계별 사용자 행동 / 시스템 반응
   - 조건부 분기(로그인 상태, 역할)
   - **접근성**: 포커스 순서, 필수 라이브 영역, 대비 목표(텍스트/대형 텍스트)
   - **카피 톤**: 존댓말/반말, 오류 메시지 톤, 금지 표현
4. **Handoff** — 다음 역할이 쓸 **화면 ID 목록**과 "비주얼 스펙에 꼭 필요한 상태 목록"을 표로 남긴다.
5. 스펙이 비면 **가정** 섹션을 두고 P0만 완성도 있게 채운다.
